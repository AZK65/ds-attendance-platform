WHAT IT DOES
------------
A Next.js web app for Qazi Driving School that manages WhatsApp groups and Zoom class attendance:
- Connects to WhatsApp via whatsapp-web.js (Puppeteer/Chromium in Docker)
- Lists WhatsApp groups and their members
- Sends group messages and private individual reminders to members about upcoming classes
- Fetches Zoom meeting participants via Zoom Server-to-Server OAuth API
- Fuzzy-matches Zoom participant names to WhatsApp group members (handles "(iPhone)" suffixes, #1120 tags, device names, possessives)
- Allows manual matching for unrecognized names, saves learned matches to DB
- Generates PDF attendance reports
- Filters out Zoom host account ("Qazi Driving School") from attendance

TECH STACK
----------
- Framework: Next.js 16.1.4, React 19.2.3, TypeScript
- Database: SQLite via Prisma ORM
- Styling: Tailwind CSS 4, Radix UI (shadcn/ui components)
- WhatsApp: whatsapp-web.js v1.34.4 with Puppeteer (Chromium in Docker)
- Zoom: Server-to-Server OAuth (ZOOM_ACCOUNT_ID, ZOOM_CLIENT_ID, ZOOM_CLIENT_SECRET)
- PDF: @react-pdf/renderer + pdfkit
- Docker: node:20-slim with Chromium, shm_size: 1gb, persistent volumes for SQLite + WhatsApp session

PRISMA MODELS
-------------
- Group: WhatsApp group info (id, name, participantCount, lastSynced)
- Contact: WhatsApp members (id, phone, name, pushName)
- AttendanceSheet: per-group attendance sheet (linked to Group)
- AttendanceRecord: individual attendance record (linked to Sheet + Contact, with status/date)
- ZoomAttendance: saved Zoom attendance results per meeting (matchedRecords, absentRecords, unmatchedZoom as JSON)
- ZoomNameMatch: learned manual matches (zoomName -> whatsappPhone/whatsappName) so next class auto-matches

ZOOM MATCHING ALGORITHM (src/lib/zoom/client.ts)
-------------------------------------------------
normalizeName() strips:
  - Parenthesized content: "Ahmed (iPhone)" -> "ahmed"
  - Bracketed content: "Ahmed [mobile]" -> "ahmed"
  - Number tags: "Kulwinder Singh #1120" -> "kulwinder singh"
  - Trailing numbers: "John 123" -> "john"
  - Possessive 's: "Ahmed's" -> "ahmed"
  - Non-alpha chars
  - Device words: iphone, ipad, android, samsung, pixel, galaxy, phone, tablet, device, mobile, macbook, laptop

matchZoomToWhatsApp() phases:
  Phase 1: Learned matches (from ZoomNameMatch DB table) — tries both raw and normalized names
  Phase 2: Exact normalized match
  Phase 3: First+Last name match (any order)
  Phase 4: First name only match (if unique)
  Phase 5: Levenshtein distance fuzzy match (threshold based on name length)
